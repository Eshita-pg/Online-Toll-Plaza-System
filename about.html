  • Inputs:
    1. User Information (Name, Car Number, Email Address, Password, Money in eWallet)
    2. Administrator Information (Email Address, Password)

  • Operations:

    • User
      1. Log-In
      2. Credit in eWallet
      3. Check eWaller Balance
      4. Log-Out

    • System
      1. Check the car number
      2. Required Fee Available Allow the car to pass Deduct money from eWallet
      3. Required Fee NOT Available Do Not Allow the car to pass Fee Payment is done manually
      4. Allow the car to pass
      5. Total Income is stored in a database

    • Administrator
      1. Log-In
      2. View transactions
      3. View total income

  • Outputs:
    1. Display day-wise transactions to administrator
    2. User can view his/her own transactions

  • Constraints:
    1. All users MUST register themselves into the system with their car numbers.
    2. A user can only view his/her own transactions
    3. Administrator can view all transactions
